# Content Batching Workflow

Separate thinking from doing. Batch by task type, not by post. This mirrors the phased production pipeline approach -- complete each stage for ALL pieces before moving to the next.

## The Content Production Pipeline

```
Ideate (30 min) -> Write (2 hrs) -> Edit (1 hr) -> Design (1 hr) -> Schedule (30 min)
```

Writing 5 posts in a row is faster than writing, editing, designing, and scheduling one post five times. Context switching is the enemy.

## Batch Approaches

| Approach | Best For | Schedule |
|----------|----------|----------|
| **Weekly batch** | 3-5 posts/week, one platform | One 3-4 hour block per week |
| **Biweekly batch** | 5-7 posts/week, multi-platform | Two 2-hour blocks per week |
| **Monthly batch** | Planned campaigns, evergreen | One full day per month + weekly touch-ups |
| **Daily creation** | Reactive/news content | 30-45 min daily (not recommended as sole approach) |

## Time-Blocking Template (Weekly Batch)

| Block | Duration | Task |
|-------|----------|------|
| **Ideate** | 30 min | Review idea bank, pick 5 topics, write hooks |
| **Draft** | 90 min | Write all 5 drafts back-to-back, no editing |
| **Edit** | 45 min | Polish all drafts, tighten hooks, check CTAs |
| **Design** | 45 min | Create all visuals, carousels, thumbnails |
| **Schedule** | 15 min | Load into scheduler with publish times |

Total: ~3.5 hours for a full week of content.

## Batching Rules

1. **Never edit while writing.** Drafting and editing use different brain modes. Switching kills speed.
2. **Use templates.** Rotate through 3-5 post formats (story, list, question, how-to, opinion). Start from structure, not blank page.
3. **Set a timer per draft.** 15-20 minutes max. If it's not flowing, move on and come back.
4. **Batch design separately.** Open Canva/Figma once, make all graphics in one session.
5. **Schedule with buffer.** Keep 2-3 posts in reserve so one bad week doesn't break the streak.

## Post Format Templates

- **Story:** Hook -> Context -> Turning point -> Lesson -> CTA
- **List:** Bold claim -> 5-7 items -> Closing insight
- **How-To:** Problem -> Steps (numbered) -> Result -> "Try this"
- **Hot Take:** Contrarian opener -> Why most get it wrong -> Your take -> Question

## Anti-Patterns

- Batching without an idea bank (60% of the session spent staring at blank screen)
- Perfectionism during drafting (write ugly first drafts, fix later)
- Skipping the schedule step (batched content in a folder never gets posted)
- No templates (starting from scratch every time)
