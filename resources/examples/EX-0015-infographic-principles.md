# Infographic Design Principles

How to turn data and processes into clear, shareable visual content.

## Data Visualization Basics

Choose the chart type by what you are communicating:

| Message | Best Chart Type |
|---------|----------------|
| Comparison between items | Horizontal bar chart |
| Change over time | Line chart |
| Part of a whole | Pie or donut chart (max 5 slices) |
| Distribution | Histogram or box plot |
| Relationship | Scatter plot |
| Process or flow | Flowchart or timeline |

Rules: never use 3D effects, always label axes, and start numerical axes at zero unless you explicitly call out the truncation.

## Visual Hierarchy for Information

Structure every infographic in this order of visual weight:

1. **Title** -- largest text, top of the graphic. One sentence max.
2. **Key statistic or hero number** -- oversized number (3-4x body text size) that anchors the story.
3. **Section headers** -- break content into 3-5 digestible chunks.
4. **Supporting data** -- charts, icons, and short text blocks within each section.
5. **Source line and branding** -- smallest text, bottom of the graphic.

## Color Coding

- **Limit to 3-5 colors.** One primary, one accent, and 1-3 supporting shades.
- **Use color to encode meaning**, not decoration. Example: green for positive metrics, red for negative.
- **Sequential palettes** (light-to-dark of one hue) for ordered data.
- **Categorical palettes** (distinct hues) for unrelated groups.
- **Accessibility:** ensure 3:1 minimum contrast between adjacent colors. Never rely on color alone to convey meaning; pair with labels or patterns.

## Icon Usage

- Use a **consistent icon set** (outlined or filled, not mixed).
- Icons replace labels, not data. One icon per concept.
- Recommended free sets: Lucide, Phosphor, Heroicons.
- Size icons uniformly (24px, 32px, or 48px grid).
- Pair every icon with a short text label for accessibility.

## Flow Direction

- **Vertical scroll** (top to bottom) for web and social media -- readers expect it.
- **Left to right** for timelines and processes.
- **Use arrows or numbered steps** to guide the eye when flow is non-obvious.
- **Z-pattern** for poster-style layouts: top-left headline, top-right image, bottom-left data, bottom-right CTA.

## Platform Sizing

| Platform | Recommended Size (px) | Orientation |
|----------|----------------------|-------------|
| LinkedIn / Twitter | 1200 x 1500 | Vertical |
| Instagram feed | 1080 x 1350 | Vertical (4:5) |
| Pinterest | 1000 x 2100 | Tall vertical |
| Blog embed | 800 x 2000+ | Vertical scroll |
| Presentation slide | 1920 x 1080 | Horizontal |

## Example Infographic Structure

A "State of Remote Work" infographic in 5 sections:

```
[TITLE BAR]  "Remote Work in 2026: By the Numbers"
[HERO STAT]  "68% of knowledge workers are hybrid or remote"
[SECTION 1]  "Work Location Breakdown" -- donut chart (remote / hybrid / office)
[SECTION 2]  "Productivity Trends" -- line chart showing output over 4 years
[SECTION 3]  "Top Challenges" -- horizontal bar chart (isolation, communication, boundaries)
[SECTION 4]  "Tools Most Used" -- icon grid with percentages (Slack, Zoom, Notion, etc.)
[SECTION 5]  "What Workers Want" -- 3 large icons with stats (flexibility, async, 4-day week)
[FOOTER]     Source: Buffer State of Remote Work 2026 | yourhandle.com
```

## Tools for Infographics

- **Canva** -- drag-and-drop templates, best for speed
- **Figma** -- best for custom layouts and team collaboration
- **Piktochart** -- purpose-built for infographics with chart widgets
- **Datawrapper** -- best for embeddable, responsive charts

## Key Points

- **One story per infographic** -- if it needs two headlines, make two infographics
- **White space is structure** -- generous padding between sections improves readability
- **Mobile test** -- preview at 400px wide before publishing; text below 14px becomes unreadable
- **Cite sources** -- unlabeled data destroys credibility
- **Shareable format** -- export as PNG for social, PDF for downloads, SVG for web embeds
