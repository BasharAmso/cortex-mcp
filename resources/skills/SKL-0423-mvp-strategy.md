## Core Concepts

An MVP is not a half-built product. It is the smallest thing you can ship that tests your core hypothesis. The goal is learning, not feature completeness.

### The MVP Litmus Test

Before building, answer three questions:
1. **What assumption am I testing?** (e.g., "People will pay for automated invoice reminders")
2. **What is the minimum I need to build to test it?** (Often less than you think)
3. **How will I measure success?** (Signups, conversions, retention, willingness to pay)

### Scope Cutting Framework

| Keep | Cut | Defer |
|------|-----|-------|
| Core value proposition | Nice-to-have features | Scalability optimizations |
| One complete user flow | Multiple user personas | Admin dashboards |
| Basic auth and data safety | Social features | Advanced analytics |
| Manual processes behind the scenes | Full automation | API integrations |

### The "Wizard of Oz" Approach

Many successful MVPs substitute manual work for automation. Zappos started by buying shoes from stores and shipping them manually. Your MVP can use spreadsheets, manual emails, or Zapier flows behind a polished front door.

### Common MVP Types

| Type | Description | Best For |
|------|-------------|----------|
| **Landing page** | Describe the product, capture emails | Demand validation |
| **Concierge** | Deliver the service manually | Service businesses |
| **Single-feature** | Build one thing well | Tool/utility products |
| **Piecemeal** | Combine existing tools (Typeform + Stripe + Notion) | Process validation |

### Timeline Discipline

Set a hard deadline (2-4 weeks for solo builders, 4-8 weeks for teams). When you hit the deadline, ship what you have. Features that did not make the cut go to the backlog. The market will tell you what to build next faster than your roadmap will.

## Key Takeaways
- An MVP tests a hypothesis, not showcases features
- Cut scope ruthlessly: one persona, one flow, one value proposition
- Manual processes behind the scenes are perfectly valid
- Set a hard ship date and honor it
- Measure learning, not feature count
