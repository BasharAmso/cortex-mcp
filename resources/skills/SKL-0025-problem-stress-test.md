# Problem Stress Test

Challenge a product or startup idea through ten structured lenses based on Uri Levine's "Fall in Love with the Problem, Not the Solution." Advisory only. The user always decides whether to proceed.

## The Ten Lenses

| Lens | Core Question |
|------|--------------|
| 1. Problem Love Test | Does the description lead with user pain or product features? |
| 2. 100 Conversations Rule | Is there evidence of real-world validation (conversations, surveys, data)? |
| 3. Problem Scale & Frequency | How many people have this problem and how often? |
| 4. Value Creation Test | Does solving this create clear, measurable value? |
| 5. Disruption Framework | How are people solving this today, and why is that unacceptable? |
| 6. Founder-Problem Fit | Is this the founder's problem, and does it extend beyond them? |
| 7. Retention Signal Test | Would users return repeatedly, or is this one-time use? |
| 8. Entrepreneurial Zone Check | Does commitment match the likely multi-year timeline? |
| 9. Narrative Test | Is the problem story more compelling than the solution description? |
| 10. Phase Discipline | Is the idea focused on one core problem, or trying to solve everything? |

## Procedure

### 1. Determine Input Richness

- **Rich:** Problem statement + target user + external validation evidence. All lenses scored.
- **Moderate:** Problem + target user, no external validation. Some lenses marked "Needs More Data."
- **Lean:** Minimal description. Only core lenses scored; total normalized.

### 2. Score Each Lens (0-2 Points)

- **Pass (2):** Strong evidence
- **Partial (1):** Present but incomplete
- **Fail (0):** Missing or inadequate

For every non-Pass lens, provide a strengthening question the founder should answer.

### 3. Determine Verdict

| Score (out of 20) | Verdict |
|-------------------|---------|
| 16-20 | **Strong** — Proceed with confidence |
| 10-15 | **Needs Work** — Address gaps before committing resources |
| 0-9 | **Weak** — Invest in problem discovery before building |

### 4. Write Report

Save to `docs/PROBLEM_STRESS_TEST.md` with lens results, top strengths, critical gaps, and recommended next steps.

## Key Rules

- Never blocks idea progression. Advisory only.
- Never fabricates evidence of validation.
- Never penalizes lean input as if it were rich input.
- Always constructive: strengthen the idea, do not reject it.
