# GDD Writing

Write a Game Design Document through conversational interview, incorporating game design theory (MDA Framework, Bartle Player Types, Self-Determination Theory). The game-development parallel to PRD Writing.

## Game Vision Challenge (Mandatory First Step)

Before any interview, establish the foundation:

1. **Core Fantasy:** What does the player *feel* while playing? (Power? Discovery? Mastery?)
2. **30-Second Loop:** What does moment-to-moment gameplay feel like? If this is not fun in isolation, nothing else matters.
3. **Differentiation:** What existing game is closest, and how is yours different?
4. **Scope Posture:** Choose one and commit:
   - EXPANSION: Dream big, explore emergent systems
   - HOLD SCOPE: Right-sized, polish the core loop
   - REDUCTION: Vertical slice, absolute minimum for the core fantasy

## Interview Rounds

| Round | Covers |
|-------|--------|
| 1. Identity & Fantasy | Genre, platform, audience, elevator pitch, unique hook |
| 2. Gameplay Loops | 30-second loop (core verb + feedback), 5-15 minute cycle, session arc, long-term progression, win/lose conditions |
| 3. Design Pillars | 3-5 falsifiable pillars, anti-pillars, MDA aesthetics ranking, Bartle type appeal, SDT motivation mapping |
| 4. Art & Audio | Visual style, tone, audio direction, game feel goals, juiciness |
| 5. Scope & Constraints | MVP features (MoSCoW), explicit exclusions, tech constraints, monetization, kill rule |

## GDD Structure (14 Sections)

1. Game Identity, 2. Core Fantasy, 3. Design Pillars, 4. Gameplay Loops, 5. Core Mechanics, 6. Progression & Economy, 7. Player Experience, 8. Art Direction, 9. Audio Direction, 10. MVP Scope, 11. NOT in Scope, 12. Risks & Assumptions, 13. Open Questions, 14. References

## Key Rules

- Never skip the core fantasy. Mechanics come after the fantasy is established.
- Never write mechanics before the 30-second loop is defined.
- Design pillars must be falsifiable. Reject vague pillars like "be fun."
- Once a scope posture is chosen, commit to it.
- Interview is conversational. Never dump all questions at once.
